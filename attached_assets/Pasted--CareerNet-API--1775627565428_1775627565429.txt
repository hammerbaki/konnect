# 데이터 정합성 최종 검증 지시서

## 목적

CareerNet API 전수 재수집 및 프론트엔드 반영이 완료되었습니다. 이 지시서는 **학과/직업 탐색**과 **진로 흥미 분석** 두 탭 간의 데이터가 정확하고 일관되게 연동되는지 최종 확인하는 작업입니다.

---

## 검증 1: 학과/직업 탐색 탭 — 샘플 기반 spot check

아래 5개 학과와 5개 직업을 실제로 조회하여 데이터가 정상 표시되는지 확인하세요. 각 항목에서 `[object Object]`, `undefined`, `null`, `NaN`, 빈 값이 표시되는 곳이 없어야 합니다.

### 학과 샘플 (다양한 계열에서 선택)

| 학과 | 확인 포인트 |
|------|-----------|
| 간호학과 | 취업률, 초임급여, 개설대학 목록, 관련 직업 연결 |
| 컴퓨터공학과 | 관련 고교 과목, 진출 분야, 홀랜드 코드 |
| 경영학과 | 관련 직업 급여 범위, 개설대학 지역 필터 |
| 한약학과 | 초임급여(399.6만원/월 — 높은 값 정상 표시 확인) |
| IT융합학과 | 관련 고교 과목에 `<br>` 태그 없이 깔끔하게 표시되는지 |

각 학과에서 확인할 것:
- 학과 설명이 표시되는가
- 취업률 숫자가 HTML 태그 없이 표시되는가
- 초임 월급여가 숫자로 표시되는가
- 관련 고교 과목이 읽을 수 있는 텍스트인가
- 관련 직업이 1개 이상 연결되어 있는가
- 개설 대학이 1개 이상 표시되는가

### 직업 샘플

| 직업 | 확인 포인트 |
|------|-----------|
| 비행기조종사 | 연봉 10,205만원 — 높은 값 정상 표시 |
| IT컨설턴트 | 관련 학과 목록 연결 |
| 간호사 | 관련 학과(간호학과)와 양방향 연결 확인 |
| 버스운전기사 | 연봉 4,000만원 — 실제 워크넷 값 (라운드넘버이지만 실제 데이터) |
| 물리학연구원 | 직업 분류(자연·생명과학직) 정확한지 |

각 직업에서 확인할 것:
- 연봉이 숫자로 표시되는가 (만원 단위)
- 관련 학과가 1개 이상 표시되는가
- 직업 설명이 있는가
- 직업 분류가 표시되는가

## 검증 2: 진로 흥미 분석 → 탐색 탭 연동

**테스트 시나리오**: 진로 흥미 분석을 1회 실행(또는 기존 결과 사용)한 뒤, 추천된 직업/학과가 탐색 탭의 데이터와 일치하는지 확인합니다.

### 확인 항목

1. **추천 직업의 급여**: 분석 결과 카드에 표시된 급여와, 해당 직업을 탐색 탭에서 조회한 급여가 **동일한 값**인가?
2. **추천 학과의 정보**: 분석 결과에 표시된 학과 설명/취업률이 탐색 탭의 해당 학과 정보와 일치하는가?
3. **추천 직업 클릭 → 탐색 상세**: 클릭하면 올바른 직업 상세로 이동하는가?
4. **추천 학과 클릭 → 탐색 상세**: 클릭하면 올바른 학과 상세로 이동하는가?
5. **"외 N개" 표시**: 추천 목록이 5개면 3개 표시 + "외 2개", 클릭 시 나머지도 정상 표시되는가?

### 교차 검증 (1건만)

추천 직업 중 1개를 골라서:
- 분석 결과에서 표시되는 급여 → 메모
- 탐색 탭에서 해당 직업 검색 → 급여 비교
- **두 값이 동일해야 함**

## 검증 3: 간단한 숫자 정합성 쿼리

아래 SQL을 실행하여 데이터 상태를 확인하세요. 결과를 로그에 출력하면 됩니다.

```sql
-- 1. 기본 현황
SELECT 
  (SELECT COUNT(*) FROM cached_jobs) as total_jobs,
  (SELECT COUNT(*) FROM cached_jobs WHERE salary IS NOT NULL) as jobs_with_salary,
  (SELECT COUNT(*) FROM cached_majors) as total_majors,
  (SELECT COUNT(*) FROM cached_majors WHERE employment IS NOT NULL) as majors_with_employment;

-- 2. 직업-학과 연결 현황
SELECT COUNT(*) as jobs_with_related_majors
FROM cached_jobs 
WHERE related_majors IS NOT NULL AND related_majors != '' AND related_majors != '[]';

-- 3. salary 단위 확인 (직업 연봉이 만원 단위인지 확인)
SELECT job_name, salary FROM cached_jobs 
WHERE job_name IN ('비행기조종사', '간호사', '버스운전기사') 
ORDER BY job_name;
-- 비행기조종사: ~10205(만원) 또는 ~102050000(원) 중 어느 형태인지 확인

-- 4. 학과 salary 단위 확인 (월급여 만원 단위)
SELECT name, salary FROM cached_majors 
WHERE name IN ('간호학과', '한약학과', '경영학과')
ORDER BY name;
```

**기대 결과:**
- total_jobs: 552
- total_majors: 510
- jobs_with_salary: 470 이상
- jobs_with_related_majors: 480 이상

숫자가 크게 벗어나면(예: jobs_with_salary < 400) 원인을 조사하세요. 근소한 차이는 일부 직업에 해당 데이터가 없는 것이므로 정상입니다.

---

## 발견된 문제 처리

- **표시 깨짐** (object Object, HTML 태그, NaN 등): 해당 렌더링 코드를 즉시 수정
- **데이터 불일치** (분석 결과와 탐색 탭 급여가 다름): 데이터 소스가 다른 테이블/필드를 참조하고 있을 가능성 → 동일 소스를 사용하도록 통일
- **데이터 누락** (특정 학과/직업에서 정보 없음): API에 해당 데이터가 없는 경우이므로 "정보 없음"으로 깔끔하게 표시되면 OK

## 완료 기준

- 위 10개 샘플(학과 5 + 직업 5)에서 표시 깨짐 없음
- 진로 흥미 분석 추천 결과와 탐색 탭 데이터 간 급여 불일치 없음
- SQL 현황 숫자가 기대 범위 이내
