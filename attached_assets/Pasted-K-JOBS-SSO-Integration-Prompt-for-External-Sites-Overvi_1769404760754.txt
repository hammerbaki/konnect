K-JOBS SSO Integration Prompt for External Sites
Overview
We need to implement SSO (Single Sign-On) integration with K-JOBS (kjobs.co.kr). When users click a service link from K-JOBS, they will be redirected to our site with a JWT token. We need to validate this token and automatically log the user in.

What K-JOBS Provides
K-JOBS sends users to our site with a JWT token in the URL:

https://konnect.careers/sso-callback?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
JWT Token Structure
The token payload contains:

{
  "userId": 123,
  "email": "user@example.com",
  "name": "사용자이름",
  "service": "konnect",  // or "kjobslink" or "diagnosis"
  "iat": 1706000000,
  "exp": 1706000300  // 5 minutes from issue time
}
Shared Secret Key
SSO_JWT_SECRET=70ee2e06ad9be842d59d9b418352da23a17b0c6d1e4ea98c97a693af45a874b0
Store this as an environment variable. NEVER expose in code.

Implementation Requirements
1. Install JWT Library
npm install jsonwebtoken
# or for TypeScript
npm install jsonwebtoken @types/jsonwebtoken
2. Create SSO Callback Endpoint
Create a route to handle incoming SSO requests:

// server/routes.ts or similar
import jwt from "jsonwebtoken";
const SSO_JWT_SECRET = process.env.SSO_JWT_SECRET;
interface SSOPayload {
  userId: number;
  email: string;
  name: string;
  service: string;
  iat: number;
  exp: number;
}
app.get("/sso-callback", async (req, res) => {
  try {
    const { token } = req.query;
    
    if (!token || typeof token !== "string") {
      return res.redirect("/login?error=missing_token");
    }
    
    if (!SSO_JWT_SECRET) {
      console.error("SSO_JWT_SECRET not configured");
      return res.redirect("/login?error=configuration_error");
    }
    
    // Verify and decode the JWT token
    let payload: SSOPayload;
    try {
      payload = jwt.verify(token, SSO_JWT_SECRET, {
        algorithms: ["HS256"]
      }) as SSOPayload;
    } catch (jwtError) {
      console.error("JWT verification failed:", jwtError);
      return res.redirect("/login?error=invalid_token");
    }
    
    // Validate service matches our site
    const expectedService = "konnect"; // or "kjobslink"
    if (payload.service !== expectedService) {
      return res.redirect("/login?error=invalid_service");
    }
    
    // Find or create user in our database
    let user = await db.query.users.findFirst({
      where: eq(users.email, payload.email)
    });
    
    if (!user) {
      // Create new user from K-JOBS data
      const [newUser] = await db.insert(users).values({
        email: payload.email,
        name: payload.name,
        kjobsUserId: payload.userId, // Optional: store K-JOBS user ID for reference
        createdAt: new Date()
      }).returning();
      user = newUser;
    }
    
    // Create session for the user
    req.session.userId = user.id;
    req.session.email = user.email;
    
    // Redirect to main page or dashboard
    res.redirect("/dashboard");
    
  } catch (error) {
    console.error("SSO callback error:", error);
    res.redirect("/login?error=server_error");
  }
});

Add to your .env or Replit Secrets:

Security Checklist
 Store SSO_JWT_SECRET as environment variable (never in code)
 Verify JWT algorithm is HS256
 Check token expiration (5 minute window)
 Validate service field matches your site
 Use HTTPS in production
 Log failed authentication attempts for security monitoring
Testing
K-JOBS will redirect to: https://your-site.com/sso-callback?token=<JWT_TOKEN>
Your site should:
Decode and verify the token
Create/find user by email
Create session
Redirect to authenticated area
Error Handling
Handle these error cases:

missing_token: No token in URL
invalid_token: JWT verification failed (expired, wrong signature, etc.)
invalid_service: Token not intended for this service
configuration_error: SSO_JWT_SECRET not set
server_error: Unexpected errors
What We Have on K-JOBS Side
On the K-JOBS side, we have:

SSO Token Generation API: POST /api/auth/sso-token

Requires authenticated user session
Accepts { service: "konnect" | "kjobslink" | "diagnosis" }
Returns { token: "jwt...", expiresIn: 300 }
Service Links: When users click external service buttons, we:

Call the SSO token API
Redirect to https://external-site.com/sso-callback?token=<token>
Shared Secret: SSO_JWT_SECRET environment variable that both sites must have
