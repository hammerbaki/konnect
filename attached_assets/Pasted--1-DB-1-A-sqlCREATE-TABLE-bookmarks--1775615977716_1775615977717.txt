지시사항: 대시보드 핵심 기능 추가 — 흥미 분석 결과 + 찜 기능

1. 찜(북마크) 기능 — DB 구조
1-A. 테이블 생성
sqlCREATE TABLE bookmarks (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  bookmark_type ENUM('university', 'major', 'job') NOT NULL,
  target_id INT NOT NULL,           -- universities.id / cached_majors.id / cached_jobs.id
  target_name VARCHAR(200) NOT NULL, -- 화면 표시용 이름 (조인 없이 빠르게 표시)
  memo TEXT,                         -- 사용자 메모 (선택)
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY unique_bookmark (user_id, bookmark_type, target_id)
);
1-B. API 엔드포인트
POST   /api/bookmarks          — 찜 추가 { bookmark_type, target_id, target_name, memo? }
DELETE /api/bookmarks/:id      — 찜 해제
GET    /api/bookmarks          — 내 찜 목록 조회 (bookmark_type별 필터 가능)
PATCH  /api/bookmarks/:id      — 메모 수정

이미 찜한 항목을 다시 POST하면 409 반환 (중복 방지)
로그인하지 않은 상태에서는 localStorage에 임시 저장하고, 로그인 시 DB로 동기화해라 (로그인 기능이 없으면 localStorage만 사용)

2. 찜 버튼 — 탐색 탭에 추가
2-A. 대학 카드

각 대학 카드 우측에 ☆ 아이콘 추가
클릭 시 ★로 변경되며 찜 등록
다시 클릭 시 찜 해제

2-B. 학과(전공) 카드

각 학과 카드 우측에 동일한 ☆ 아이콘 추가
찜 등록/해제 동작 동일

2-C. 직업 카드

각 직업 카드 우측에 동일한 ☆ 아이콘 추가
찜 등록/해제 동작 동일

2-D. 진로 흥미 분석 결과 화면의 추천 카드

추천 직업 카드, 추천 학과 카드에도 ☆ 아이콘 추가
분석 결과에서 바로 찜할 수 있도록 해라

3. 대시보드 상단 — 핵심 기능 영역
현재 대시보드 최상단에 다음 두 개의 영역을 추가해라. 기존 대시보드 콘텐츠 위에 배치한다.
3-A. 진로 흥미 분석 결과 요약 카드
┌──────────────────────────────────────────┐
│  🧠 나의 흥미 분석 결과                    │
│                                          │
│  [분석 완료 시]                            │
│  상위 흥미: 과학·탐구 (85점), IT (80점),   │
│           공학·기술 (75점)                 │
│  상위 역량: 수리·논리력, 창의력             │
│                                          │
│  추천 직업: IT컨설턴트, 데이터분석가 외 3개  │
│  추천 학과: 컴퓨터공학과, 경영정보과 외 3개  │
│                                          │
│  [상세 결과 보기 →]  [다시 분석하기]        │
│                                          │
│  [분석 미완료 시]                          │
│  아직 흥미 분석을 하지 않았습니다.           │
│  30문항 · 약 5분이면 나에게 맞는            │
│  학과와 직업을 찾을 수 있습니다.            │
│                                          │
│  [분석 시작 →]                             │
└──────────────────────────────────────────┘
데이터 출처:

aptitude_analyses 테이블에서 해당 사용자의 최신 분석 결과를 가져와라
분석 결과가 없으면 "미완료" 상태를 표시
"상세 결과 보기" 클릭 → 진로 흥미 분석 결과 페이지로 이동
"다시 분석하기" 클릭 → 진로 흥미 분석 시작 페이지로 이동

규칙:

각 항목 클릭 → 탐색 탭에서 해당 항목 상세 페이지로 이동
✕ 버튼 → 찜 해제 (확인 팝업 표시: "관심 목록에서 제거하시겠습니까?")
학과 항목에는 관련 직업 급여 범위를 함께 표시 (이미 구현된 관련 직업 급여 범위 로직 재사용)
직업 항목에는 연평균 급여를 표시 (DB salary 값)
대학 항목에는 지역과 설립구분을 표시

두 카드를 가로로 나란히 배치 (데스크탑)
모바일에서는 세로로 쌓이도록 반응형 처리
두 카드의 높이를 동일하게 맞춰라

4. 데이터 정확성 원칙

대시보드에 표시되는 모든 데이터는 DB 원본에서 가져와라
찜 목록의 급여, 분류 등은 cached_jobs, cached_majors, universities 테이블에서 실시간 조회해라
LLM으로 대시보드 요약이나 찜 항목 설명을 생성하지 마라
데이터가 없는 필드는 숨겨라
