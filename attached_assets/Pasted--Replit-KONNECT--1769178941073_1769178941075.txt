너는 Replit에서 실행 가능한 웹앱 코드를 수정/추가하는 시니어 풀스택 개발자야.
목표: KONNECT “구직자 프로필” 화면에 아래 항목(이미지에 있는 섹션들)을 추가하고, 여러 개를 등록/수정/삭제할 수 있도록 CRUD까지 완성해줘.
기존 디자인/컴포넌트 스타일(입력박스, 드롭다운, 버튼: 추가/저장/취소)을 유지하고, 폼 유효성검사/에러표시/저장 성공 토스트까지 포함해.

1) 작업 범위
- 구직자 프로필 페이지에 신규 섹션 4개 추가
  A. 어학(외국어) 시험/역량
  B. 자격증/면허
  C. 수상/공모전
  D. 추천인(레퍼런스)
- 각 섹션은 “목록 + 추가 폼(모달 또는 인라인)” 형태로 구현
- 각 항목은 복수 등록 가능(배열), 항목별 수정/삭제 가능
- 저장 시 DB(또는 기존 저장소)에 반영되고 새로고침해도 유지되게 구현
- 프론트/백엔드/DB 스키마/마이그레이션(또는 모델)까지 한 번에 반영

2) UI 요구사항(공통)
- 섹션 제목 + 구분선(이미지처럼) + 입력 폼 + 우측 하단 버튼(취소/저장)
- 목록에서는 각 항목을 카드 또는 테이블로 보여주고, 항목 우측에 수정/삭제 버튼 제공
- 필수값 누락 시 필드 하단에 에러 메시지(짧게)
- 날짜는 date picker 또는 YYYY-MM-DD 입력 지원
- 드롭다운은 검색 가능(가능하면)

3) 데이터 모델(권장 스키마)
- JobSeekerProfile 테이블(또는 문서)에서 아래를 배열로 저장
  - languageTests: LanguageTest[]
  - licenses: LicenseItem[]
  - awards: AwardItem[]
  - references: ReferenceItem[]

4) 필드 정의(이미지 기반, 최소 이 필드들은 반드시 구현)

A. 어학(외국어) 시험/역량(languageTests)
- examName (드롭다운, 예: TOPIK, TOEIC, TOEFL, IELTS, JLPT, HSK, 기타)
- scoreType (드롭다운: 등급, 점수)
- scoreValue (텍스트 또는 숫자)
- acquiredDate (날짜)
- isPending (체크박스: 준비중/응시예정)
- expiryDate (날짜, 선택)
- note (텍스트, 선택)

B. 자격증/면허(licenses)
- category (드롭다운: 자격증, 면허)
- licenseType (드롭다운 또는 텍스트: 종류)
- title (텍스트: 자격/면허명)
- issuer (텍스트: 발급기관)
- status (드롭다운: 취득, 준비중, 만료)
- acquiredDate (날짜, 선택)
- licenseNumber (텍스트, 선택)
- expiryDate (날짜, 선택)

C. 수상/공모전(awards)
- awardType (드롭다운: 수상, 공모전, 대회, 기타)
- rank (드롭다운/텍스트: 대상/최우수/우수/장려/참가/기타)
- titleAndHost (텍스트: 상명 또는 공모전명 + 주최기관명)
- awardDate (날짜)
- year (숫자, 선택) / month (숫자, 선택)  ※ UI에서는 awardDate로 충분하면 year/month는 자동 파생해도 됨
- note (텍스트, 선택)

D. 추천인(레퍼런스)(references)
- relation (드롭다운: 교수, 직장상사, 동료, 지인, 기타)
- name (텍스트: 추천인 성명)
- organization (텍스트: 소속/기관, 선택)
- phone (텍스트: 연락처)
- email (텍스트: 이메일, 선택)
- note (텍스트, 선택)

5) API/저장 로직
- 기존에 프로필 저장 API가 있으면 해당 API에 위 4개 배열 필드 포함하도록 확장
- 없다면 아래 엔드포인트를 추가
  - GET /api/profile/jobseeker/:id
  - PUT /api/profile/jobseeker/:id  (전체 프로필 저장)
  - 또는 섹션별:
    - POST /api/profile/jobseeker/:id/language-tests
    - POST /api/profile/jobseeker/:id/licenses
    - POST /api/profile/jobseeker/:id/awards
    - POST /api/profile/jobseeker/:id/references
    - PUT/DELETE도 동일 패턴
- 서버 측 검증(필수값 체크) 포함

6) 구현 가이드(기술 스택 자동 파악)
- 현재 레포의 프레임워크(Next/React/Vite/Express 등)를 먼저 확인하고, 그 방식대로 구현
- 폼 라이브러리(React Hook Form 등)가 이미 있으면 그대로 사용
- DB가 Prisma면 Prisma 스키마 + 마이그레이션 추가
- Supabase/Firebase면 해당 방식으로 컬럼/문서 구조 업데이트
- 타입스크립트 사용 중이면 타입 정의까지 포함

7) 완료 조건(반드시)
- 구직자 프로필 화면에서 4개 섹션이 보이고, 추가/수정/삭제/저장이 동작
- 저장 후 재접속해도 데이터 유지
- 빈 값/형식 오류에 대한 에러 메시지 노출
- 시드 데이터 1~2개를 예시로 넣어 로컬에서 바로 확인 가능(가능하면)

8) 산출물
- 수정/추가된 파일 목록을 마지막에 요약
- 실행 방법(필요 시 마이그레이션/환경변수)도 함께 안내

지금 바로 코드베이스를 스캔한 뒤 위 요구사항을 구현해줘.
먼저 현재 프로젝트 구조를 파악하고(라우팅/프로필 컴포넌트/저장 로직 위치), 가장 적은 변경으로 확장해.
