# 대시보드 "외 N개" 인라인 펼치기 개선 지시서

## 현재 문제

대시보드 "나의 흥미 분석 결과" 섹션에서:
- 추천 직업/학과 **칩 클릭** → 해당 직업/학과 상세 페이지로 이동
- **"외 2개" 클릭** → 흥미 분석 결과 페이지로 이동

칩과 "외 2개"의 클릭 동작이 불일치합니다. 사용자는 "나머지 2개도 보여주겠지"라고 기대하지만 갑자기 페이지가 전환됩니다.

## 변경 내용

"외 N개"를 클릭하면 **페이지 이동 없이 그 자리에서 나머지 칩이 펼쳐지도록** 변경합니다.

### 동작 변경

```
[변경 전]
추천 직업
[물리학연구원] [경제학연구원] [언어학연구원]  외 2개  ← 클릭 시 분석 결과 페이지로 이동

[변경 후 — 접힌 상태]
추천 직업
[물리학연구원] [경제학연구원] [언어학연구원]  외 2개  ← 클릭 시 아래처럼 펼침

[변경 후 — 펼친 상태]
추천 직업
[물리학연구원] [경제학연구원] [언어학연구원] [데이터분석가] [시스템엔지니어]  접기
```

- 모든 칩은 동일하게 **클릭 = 해당 상세 페이지 이동**
- 전체 분석 결과 페이지는 기존 **"상세 결과 보기 →"** 버튼이 담당 (역할 분리)

### 구현

```tsx
// 추천 직업 섹션 예시
const [showAllJobs, setShowAllJobs] = useState(false);
const INITIAL_COUNT = 3;

const visibleJobs = showAllJobs 
  ? recommendedJobs 
  : recommendedJobs.slice(0, INITIAL_COUNT);
const hiddenCount = recommendedJobs.length - INITIAL_COUNT;

return (
  <div>
    <p className="text-sm text-gray-500 mb-2">추천 직업</p>
    <div className="flex flex-wrap gap-2 items-center">
      {visibleJobs.map(job => (
        <button
          key={job.id}
          onClick={() => navigate(`/explore/jobs/${job.name}`)}
          className="px-3 py-1.5 rounded-full bg-purple-100 text-purple-700 text-sm hover:bg-purple-200 transition-colors"
        >
          {job.name}
        </button>
      ))}
      
      {hiddenCount > 0 && !showAllJobs && (
        <button
          onClick={() => setShowAllJobs(true)}
          className="text-sm text-gray-500 hover:text-gray-700 underline"
        >
          외 {hiddenCount}개
        </button>
      )}
      
      {showAllJobs && hiddenCount > 0 && (
        <button
          onClick={() => setShowAllJobs(false)}
          className="text-sm text-gray-500 hover:text-gray-700 underline"
        >
          접기
        </button>
      )}
    </div>
  </div>
);

// 추천 학과 섹션도 동일한 패턴 적용 (별도 state: showAllMajors)
```

### 주의사항

1. 추천 직업과 추천 학과 각각 **별도 state**로 관리 (하나를 펼쳐도 다른 쪽에 영향 없음)
2. 추천 항목이 3개 이하면 "외 N개" 자체가 표시되지 않음
3. "접기" 클릭 시 다시 3개만 표시되는 원래 상태로 복귀
4. 기존 "상세 결과 보기 →" 버튼은 그대로 유지
