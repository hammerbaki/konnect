아래 설명을 바탕으로 두 개의 메뉴 페이지를 구현해줘.
## 이미 준비된 것
PostgreSQL DB에 아래 테이블과 데이터가 들어가 있어 (data.sql로 import 완료):
- cached_majors (235행): 학과 정보
- cached_jobs (552행): 직업 정보  
- university_info (489행): 대학 정보
---
## 메뉴 1: 학과/직업 탐색
### DB 테이블 스키마
cached_majors:
  id, major_name(학과명), category(계열), description(설명),
  related_jobs(JSON 배열), related_subjects(JSON 배열),
  universities(JSON 배열 - 각 원소: {univName, region, type, competition, employmentRate, tuition(천원단위), scholarship(만원단위)}),
  holland_code, demand(수요전망), employment_rate(학과취업률%)
cached_jobs:
  id, job_name(직업명), field(직업분야), description(설명),
  related_majors(JSON 배열), salary(월평균임금 만원),
  growth(취업전망: 매우좋음/좋음/보통/나쁨/매우나쁨),
  qualifications(JSON 배열), holland_code
university_info:
  id, univ_name, region, univ_type(대학교/전문대 등),
  foundation_type(국공립/사립), competition_rate(경쟁률),
  employment_rate(취업률%), avg_tuition(평균등록금 천원단위),
  scholarship_per_student(1인당장학금 원단위)
### API 엔드포인트 (Express)
GET /api/explore/majors?search=&category=&page=1&limit=20
  → cached_majors에서 검색/필터링, universities JSON 포함 반환
GET /api/explore/jobs?search=&field=&page=1&limit=20
  → cached_jobs에서 검색/필터링 반환
GET /api/explore/universities?search=&region=&page=1&limit=20
  → university_info에서 검색/필터링 반환
GET /api/explore/categories
  → cached_majors의 category 목록, cached_jobs의 field 목록 반환
### 프론트엔드 UI
탭 3개: 학과 | 직업 | 대학
학과 탭:
- 상단: 텍스트 검색창 + 계열 드롭다운 필터
- 카드 목록: 학과명, 계열 배지, 설명(2줄 말줄임), 관련직업 태그
- 카드 클릭 시 확장: 개설대학 목록(대학명, 지역, 경쟁률, 대학취업률, 등록금), 관련과목
- 주의: 취업률은 학과별이 아닌 "대학 전체 취업률"이므로 "대학취업" 레이블 표시
직업 탭:
- 상단: 텍스트 검색창 + 직업분야 드롭다운 필터
- 카드 목록: 직업명, 분야 배지, 설명(2줄), 월평균 X만원, 취업전망(좋음/보통/나쁨 색상 구분)
- 카드 클릭 시 확장: 관련학과 태그, 필요자격증/면허
대학 탭:
- 상단: 텍스트 검색창 + 지역 드롭다운 필터
- 카드 목록: 대학명, 지역, 설립유형(국공립/사립), 경쟁률, 취업률%, 평균등록금(천원÷10=만원으로 표시)

## 메뉴 2: 전공 적성 분석
### DB 테이블 스키마 (직접 생성 필요)
aptitude_analyses:
  id(serial PK), user_id(varchar), 
  interest_scores(JSON: {SCI,ENG,MED,BIZ,LAW,EDU,ART,IT,SOC} 각 0-100),
  aptitude_scores(JSON: {VERBAL,MATH,SPATIAL,CREATIVE,SOCIAL,SELF} 각 0-100),
  recommended_jobs(JSON 배열: [{name,reason,salaryRange,outlook}]),
  recommended_majors(JSON 배열: [{name,category,reason}]),
  summary(text), created_at
### 30개 문항 구성
- 1~18번: 흥미 문항 (각 9개 카테고리: SCI/ENG/MED/BIZ/LAW/EDU/ART/IT/SOC에서 2문항씩)
  예) "자연현상이나 과학적 원리에 호기심이 많다" → SCI
- 19~30번: 적성 문항 (각 6개 능력: VERBAL/MATH/SPATIAL/CREATIVE/SOCIAL/SELF에서 2문항씩)
  예) "글쓰기나 말하기로 내 생각을 잘 표현할 수 있다" → VERBAL
- 각 문항은 1~5점 리커트 척도
### API 엔드포인트
GET /api/aptitude/questions → 30개 문항 배열 반환
POST /api/aptitude/analyze
  body: { answers: [{questionId, score}] }
  → 흥미/적성 점수 계산 후 GPT로 직업/학과 추천 생성
  → DB 저장 후 결과 반환
GET /api/aptitude/latest → 최근 분석 결과 반환
### GPT 프롬프트 (gpt-4o-mini 사용)
cached_jobs에서 상위 20개 직업명, cached_majors에서 상위 20개 학과명을 쿼리해서
컨텍스트로 제공 후 점수 기반 추천 (RAG 방식)
### 프론트엔드 UI
3단계 화면 전환:
1. 시작 화면: 검사 설명 + 시작 버튼
2. 문항 화면: 프로그레스바 + 현재 문항 + 1~5점 선택 버튼, 이전/다음 내비게이션
3. 결과 화면:
   - 흥미 레이더 차트 (9개 카테고리)
   - 적성 바 차트 (6개 능력)
   - 추천 직업 카드 3개 (직업명, 추천이유, 예상연봉)
   - 추천 학과 카드 3개 (학과명, 계열, 추천이유)
   - 다시 검사하기 버튼

## 공통 기술 스택
- Backend: Node.js + Express + PostgreSQL (pg 또는 drizzle-orm)
- Frontend: React + Vite + TailwindCSS
- AI: OpenAI gpt-4o-mini (OPENAI_API_KEY 환경변수)
- 차트: recharts 라이브러리
## 주의사항
- 등록금 표시: avg_tuition은 천원 단위이므로 ÷10 하여 만원으로 표시
- scholarship_per_student는 원 단위이므로 ÷10000 하여 만원으로 표시
- cached_jobs의 취업전망(growth) 값: "매우좋음", "좋음", "보통", "나쁨", "매우나쁨"
- universities JSON 배열 내 employmentRate는 학과 취업률이 아닌 대학 전체 취업률임
