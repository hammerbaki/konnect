목표: 기존 프로필 타입(초등/중등/고등/대학/구직자)에 “외국인유학생” 프로필을 신규 추가하고,
아래 ‘외국인 취업 지원용 이력서 서식’의 섹션/필드를 그대로 반영한 프로필 입력 화면 + 저장/조회까지 end-to-end로 완성해줘. :contentReference[oaicite:0]{index=0}

[0) 사전 확인(먼저 스캔)]
- 프로필 타입/유형을 어디서 관리하는지 찾기 (enum/constants)
- 프로필 데이터 저장 구조 확인 (DB 스키마/Prisma/Supabase/Firebase/REST API 등)
- 기존 프로필(대학생/구직자) 폼 구조 확인 (공통 폼인지, 타입별 컴포넌트인지)

[1) 프로필 타입 추가]
- profileType(또는 userType)에 아래 항목 추가
  - key: INTERNATIONAL_STUDENT
  - label: 외국인유학생
- 프로필 선택 UI(사이드바/드롭다운/탭)에 “외국인유학생” 노출
- 선택 후 해당 프로필 입력 화면으로 진입 가능해야 함

[2) 외국인유학생 프로필 화면 구현(Resume 기반)]
- 신규 화면/컴포넌트: InternationalStudentProfileForm
- UI는 기존 프로필 폼 디자인 규칙을 그대로 사용(입력, 섹션 헤더, 구분선, 저장/취소 버튼)
- 섹션 구성은 아래와 동일(서식 기준)

2-1) 기본 정보(Basic Information)
- name (이름)
- nationality (국적)
- gender (성별)
- dateOfBirth (생년월일)
- phone (연락처)
- email (이메일)
- address (주소)

2-2) 체류 정보(Visa Information)
- currentVisaType (현재 비자 유형)  예: D-2, D-4, D-10, F-2, F-5, 기타
- visaExpiryDate (비자 만료일)
- availableWorkType (근무 가능 형태)  예: 정규직/계약직/인턴/아르바이트
- partTimeHoursPerWeek (취업 가능 시간-파트타임)  예: 주 n시간 이내 (선택)

2-3) 희망 근무 조건(Job Preference)
- desiredPosition (희망 직무)
- preferredLocation (희망 근무 지역)
- expectedSalary (희망 급여)  (숫자+단위 또는 텍스트)
- availableStartDate (근무 가능 시작일)

2-4) 학력 사항(Education)  ※ 복수 등록(CRUD)
- education[] 배열로 저장
  - schoolName (학교명)
  - schoolCountry (국가)
  - major (전공)
  - degree (학위)
  - periodStart (시작일)
  - periodEnd (종료일)
  - isEnrolled (재학중 여부)  ※ true면 periodEnd 비활성/숨김

2-5) 경력 사항(Work Experience)  ※ 복수 등록(CRUD)
- experienceStatus (경력 구분)  예: 신입 / 경력 / 관련 경력 없음
- workExperiences[] 배열
  - companyName (회사명)
  - companyCountry (국가)
  - positionAndRole (직무 및 역할)
  - periodStart (시작일)
  - periodEnd (종료일)
  - mainTasksAndAchievements (주요 업무 및 성과)

2-6) 언어 능력(Language Skills)
- nativeLanguage (모국어)
- koreanLevel (한국어 수준)  Advanced/Intermediate/Beginner (상/중/하)
- topikLevel (TOPIK 급수) (선택)
- englishLevel (영어 수준)  Advanced/Intermediate/Beginner (상/중/하)
- englishTestName (영어 시험명: TOEIC/TOEFL/IELTS/기타) (선택)
- englishTestScore (시험 점수) (선택)
- otherLanguages (기타 언어)  (텍스트 또는 태그/배열)

2-7) 보유 역량 및 기술(Skills & Competencies)
- jobRelatedSkills (직무 관련 기술)
- computerItSkills (컴퓨터/IT 활용 능력)
- certificates (자격증)  (텍스트 또는 기존 자격증 섹션과 연결 가능)

2-8) 자기소개(Short Introduction)
- reasonForKoreaEmployment (한국 취업을 희망하는 이유)
- strengthsAndPersonality (본인의 강점 및 성격)
- contributionToCompany (회사에 기여할 수 있는 부분)

2-9) 기타 사항(Additional Information)
- availableDaysHours (근무 가능 요일/시간)
- specialNotes (특이사항)

2-10) 사실확인(선택)
- certifyTrue (체크박스) “기재 내용이 사실과 다름없음 확인”
- dateSigned (작성일) (선택)
- applicantSignatureName (서명 이름) (선택)

[3) 저장/조회(필수: end-to-end)]
- 외국인유학생 프로필 데이터가 DB에 저장되고, 새로고침/재로그인 후에도 유지되어야 함
- 기존 프로필 저장 API/저장 로직을 확장해서 INTERNATIONAL_STUDENT도 동일하게 동작
- 서버/클라이언트 검증에서 profileType 허용값에 INTERNATIONAL_STUDENT 추가

[4) 유효성(최소)]
- 필수값: name, nationality, phone, email, currentVisaType, desiredPosition, schoolName(학력 1개 이상)
- 이메일/전화번호 간단 형식 검증
- 저장 실패/성공 안내(토스트 또는 메시지)

[5) 결과물 정리]
- 변경/추가된 파일 목록 요약
- 어디를 수정했는지 체크리스트(프로필 타입 / UI / 라우팅 / 저장 / DB)
- 실행 방법(마이그레이션 필요 시 명령어 포함)

[구현 원칙]
- 기존 구조/컴포넌트/스타일을 최대한 재사용하고 “최소 변경”으로 구현
- 학력/경력은 반드시 복수 등록(CRUD) 가능해야 함 (추가/수정/삭제)
- 타입스크립트면 타입 정의까지 업데이트
- 가능하면 예시 데이터 1개 시드로 바로 확인 가능하게
